## 在线问卷系统测试报告

### 自动化测试结构（2026-04-07 更新）

后端自动化测试已统一迁移到 `backend/test` 下按模块分组：

- `backend/test/service`
- `backend/test/migration`
- `backend/test/repository/mongo`
- `backend/test/delivery/http`

执行方式：在 `backend/` 目录运行 `go test ./...`。

说明：`internal/**` 下已不再放置测试文件，相关白盒测试通过测试桥接导出保持可测性。

下面我将对照**初始用户需求**，详细展示**测试过程**

报告中引用的所有图片均为测试过程中的**真实截图**

#### 一、注册和登录

用户输入用户名和密码，点击注册即可：

![1](./2.png)

  密码有强度要求，至少8位，如下图，就提示无法注册：

![1](./1.png)

  注册成功后提示登录：

![1](./3.png)

登录账号：

![1](./4.png)

#### 二、创建问卷

新用户显然是没有创建过问卷的：

![1](./5.png)

点击创建问卷，跳转到这个界面：

![1](./6.png)

可以输入问卷标题和问卷说明，可以勾选允许匿名填写，这样发布者就无法看到填写者的用户名：

![1](./7.png)

点击添加题目，可以输入题目标题，选择题型，勾选是否为必答题：

![1](./8.png)

示例一：单选题

点击添加选项可以不断添加新的选项，点击选项右边的❌️，可以删除这个选项

![1](./9.png)

示例二：多选题

可以设置最少选择几个选项，最多选择几个选项

![1](./10.png)

示例三：文本填空

可以设置最少字数和最多字数

![1](./11.png)

示例四：数字填空

可以设置填写数字的最小值和最大值，以及是否必须为整数

![1](./12.png)

出完题后，可以设置跳转规则，下拉列表选择源题目ID

![1](./13.png)

可以选择条件是等于还是包含，还是大于或小于

![1](./14.png)

可以设置条件值，如果是选择题，就是一个下拉列表

![1](./15.png)

如果是数字题，就是一个标准值，等于这个值才会跳转

（当然判定条件也可以是大于或小于，取决于前面选的什么）

![1](./17.png)

可以设置跳转到哪个题目

![1](./16.png)

点击保存问卷，提示创建成功

![1](./18.png)

刚创建的问卷会被保存为草稿

![1](./19.png)

点击发布可以在图形化界面中设置截止时间

![1](./20.png)

![1](./21.png)

再点击确认发布就会正式发布，可以随时关闭，可以查看统计结果，可以复制链接

![1](./22.png)

只要点击就会自动复制到剪贴板，无需手动框选

![1](./23.png)

访问这个链接就可以填写问卷了
